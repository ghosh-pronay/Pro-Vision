## Why this matters

`npm audit` reports high-severity vulnerabilities in transitive dependencies: `ws` (memory exhaustion DoS) via `convex`, and `dompurify` (attribute injection) via convex or firebase. These are known CVEs with available patches.

## Current state

- `package.json:22` — `convex@^1.41.0` depends on vulnerable `ws`
- `npm audit` confirms 2 high + 1 moderate advisories
- `functions/` has no `node_modules/` (dependencies not installed)

## Steps

### Step 1: Run npm audit to identify issues

```bash
npm audit
```

Note the high/critical advisories.

**Verify**: Command runs successfully

### Step 2: Run npm audit fix

```bash
npm audit fix
```

This applies non-breaking updates to fix vulnerabilities.

**Verify**: `npm audit` → reduced high/critical count

### Step 3: Add overrides if needed

If `ws` or `dompurify` can't be fixed via `npm audit fix`, add overrides to `package.json`:

```json
"overrides": {
  "ws": ">=8.20.2",
  "dompurify": ">=3.4.11"
}
```

Then run `npm install` to apply.

**Verify**: `npm audit` → no high/critical

### Step 4: Install functions dependencies


**Verify**: `ls functions/node_modules/` → directory exists
